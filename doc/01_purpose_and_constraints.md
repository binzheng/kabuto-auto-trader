# 日本株自動売買システム - 目的・前提・制約・非対象

## 1. 目的

本システムは、TradingView によるテクニカル分析に基づくシグナルを起点として、日本株式市場における売買注文を全自動で執行することを目的とする。個人投資家が手動介入なしに、事前定義された売買戦略を24時間体制で運用可能とし、機会損失の最小化と執行精度の向上を実現する。


## 2. 前提条件

### 2.1 技術基盤
- **シグナル生成**: TradingView のアラート機能を用いて売買シグナルを生成
- **中継サーバー**: macOS または Linux 環境で稼働する中継サーバー（Webhook 受信・コマンド転送）
- **執行環境**: Windows 仮想マシン（VM）上で Excel + MarketSpeed II RSS を稼働
- **証券会社**: MarketSpeed II（楽天証券）の RSS 機能を利用可能であること

### 2.2 運用体制
- 個人投資家による自己資金での運用を想定
- システム監視は投資家本人が実施
- 証券口座は投資家本人名義のものを使用

### 2.3 法的要件
- 金融商品取引法における自己売買の範囲内での運用
- 第三者資金の運用・助言業務には該当しない個人利用
- 証券会社の API 利用規約・自動売買に関する規約への準拠


## 3. 制約条件

### 3.1 技術的制約
- **macOS ネイティブ実行不可**: MarketSpeed II が Windows 専用のため、macOS での直接実行は不可能
- **Windows VM 必須**: macOS 環境では Parallels Desktop / VMware Fusion 等による Windows VM が必須
- **RSS 機能の限界**: MarketSpeed II RSS の仕様による取得可能データ・実行可能操作の制限
- **ネットワーク依存**: TradingView → 中継サーバー → Windows VM の通信経路が全て稼働している必要

### 3.2 運用上の制約
- **市場時間**: 日本株式市場の取引時間（9:00-11:30, 12:30-15:00）に限定
- **注文制限**: 証券会社による1日あたりの注文回数・金額上限
- **リスク管理**: ストップロス・ポジションサイズの上限を事前設定必須
- **システム障害時**: 手動介入によるポジション管理手段の確保が必要

### 3.3 法的・コンプライアンス制約
- **本人名義口座**: 他人名義での売買は金融商品取引法違反
- **相場操縦の禁止**: 見せ玉・仮装売買等の不公正取引に該当する戦略は禁止
- **ログ保存**: 税務調査・トラブル時の証跡として全取引ログを保存
- **証券会社規約**: 自動売買の利用について証券会社の承認または黙認範囲内であることの確認


## 4. 非対象範囲

### 4.1 本システムが実施しない事項
- **ファンダメンタル分析**: 決算情報・ニュース等の自動解析は含まない
- **複数証券会社対応**: MarketSpeed II（楽天証券）以外の証券会社システムへの連携
- **信用取引・先物オプション**: 現物株式のみを対象（拡張可能だが初期対象外）
- **ポートフォリオ最適化**: 複数銘柄間のリバランス機能は含まない

### 4.2 責任範囲外
- **投資成果**: システムの正常動作と投資成果は無関係であり、損失の責任は利用者が負う
- **市場リスク**: 急激な相場変動・流動性欠如による執行不能
- **システム障害**: 証券会社システム・TradingView サービスの障害による機会損失
- **税務申告**: 確定申告は投資家本人の責任で実施

### 4.3 将来的な拡張候補（現時点では非対象）
- 複数戦略の並行運用とパフォーマンス比較機能
- 機械学習による戦略パラメータの自動最適化
- モバイルアプリによるリアルタイム監視機能
- バックテスト環境との統合

## 5. 補足：運用開始前の確認事項

- 楽天証券における自動売買利用規約の最新版確認
- 少額・低頻度での実運用テスト期間の設定（最低1ヶ月推奨）
- 緊急停止手順（Windows VM シャットダウン、TradingView アラート無効化等）の文書化
- バックアップ戦略（Excel ファイル、設定ファイル、ログファイルの定期保存）
