## 異常時の対応（発生時のみ）

### Kill Switch 発動時

| 手順 | チェック |
|------|---------|
| 1. AuditLog で発動理由確認 | ☐ |
| 2. 発動理由の妥当性判断（5連続損失/日次損失/-50,000円/異常頻度） | ☐ |
| 3. 原因分析（戦略 or システム） | ☐ |
| 4. 対策実施（戦略修正 or 設定変更） | ☐ |
| 5. 再開判断・実施 | ☐ |

### API接続断時

| 手順 | チェック |
|------|---------|
| 1. VPS サーバー稼働確認 | ☐ |
| 2. サーバー再起動（必要時） | ☐ |
| 3. CheckAPIConnection() で再接続確認 | ☐ |

### RSS接続断時

| 手順 | チェック |
|------|---------|
| 1. MarketSpeed II 起動確認 | ☐ |
| 2. MarketSpeed II 再起動 | ☐ |
| 3. RSS 有効化確認 | ☐ |
| 4. CheckRSSConnection() で再接続確認 | ☐ |
